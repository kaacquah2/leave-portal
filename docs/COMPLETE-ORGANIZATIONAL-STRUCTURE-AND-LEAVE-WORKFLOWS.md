- **Legal Unit** - Head acts as HoD
- **Public Relations / Communications Unit** - Head acts as HoD

### WORKFLOW 1: Standard Employee (Unit-Based Staff)

**Position**: Employee in a Core Directorate Unit  
**Example**: Accountant in Finance / Accounts Unit

```

  â†“
```

### WORKFLOW 3: Immediate Supervisor Leave

**Position**: Immediate Supervisor / Senior Officer

## âœ… Compliance Rules

### Rule 1: Self-Approval Prevention

**Enforcement**: System automatically prevents self-approval

**Examples**:
- âŒ Supervisor cannot approve their own leave
- âŒ Unit Head cannot approve their own leave
- âŒ Director cannot approve their own leave
- âŒ HR Officer cannot validate their own leave
- âŒ Chief Director cannot approve their own leave

### Rule 2: Acting Officer Requirement

**Required For**:
- âœ… Unit Heads
- âœ… Directors
- âœ… HR Directors
- âœ… Heads of Independent Units
- âœ… Critical positions (Internal Audit, Legal)

**Enforcement**: System blocks approval if acting officer not assigned

### Rule 3: HR Validation Mandatory

**Required For**: All leave requests (except Chief Director)

**Enforcement**: System blocks final approval if HR validation not completed

**Exception**: HR Officer's own leave (validated by HR Director)

### Rule 4: Sequential Approval

**Enforcement**: System prevents skipping approval levels

**Example**: Cannot approve Level 3 if Level 2 is not approved

### Rule 5: External Clearance (PSC/OHCS)

**Required For**:
- Study Leave (with or without pay)
- Leave of Absence
- Secondment
- Leave exceeding statutory limits

**Enforcement**: System blocks final approval if external clearance not approved

### Rule 6: Balance Validation

**Enforcement**: System validates leave balance at each approval level

**Final Check**: HR Officer validates balance before final approval

## ğŸ” Workflow Determination Logic

The system automatically determines the correct workflow based on:

1. **Staff Position**: Director, Unit Head, Supervisor, Employee
2. **Organizational Unit**: Core Directorate or Independent Unit
3. **Directorate**: Which directorate staff belongs to
4. **Role**: System role assignment

**Priority Order**:
1. Check if Director â†’ Use Director workflow
2. Check if Unit Head â†’ Use Unit Head workflow
3. Check if HRMD staff â†’ Use HRMD workflow
4. Check if Independent Unit â†’ Use Independent Unit workflow
5. Default â†’ Use Standard Employee workflow

## ğŸ“ Implementation Notes

### Database Fields Used

- `StaffMember.position` - Determines if Director/Unit Head
- `StaffMember.unit` - Determines unit assignment
- `StaffMember.directorate` - Determines directorate
- `StaffMember.immediateSupervisorId` - Determines supervisor
- `StaffMember.actingOfficerId` - Required for critical positions

### Approval Step Tracking

Each approval level is tracked in `ApprovalStep` table:
- `level` - Sequential level (1, 2, 3, ...)
- `approverRole` - Required role for approval
- `status` - pending | approved | rejected
- `previousLevelCompleted` - Enforces sequential approval

### Compliance Validation

Before each approval:
1. Check self-approval prevention
2. Check acting officer assignment (if required)
3. Check leave balance
4. Check HR validation (if applicable)
5. Check external clearance (if applicable)
6. Check sequential approval order
